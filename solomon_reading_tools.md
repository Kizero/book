# 工具书提取引擎 · Solomon视角 · v1.0
> 与批判性阅读引擎（solomon_reading_v4）配套使用。
> 先判断书的类型，再选引擎。话语体系书用批判引擎，工具书用这个。

---

## 两个引擎的根本区别

| | 批判性阅读引擎 | 工具书提取引擎 |
|--|--------------|--------------|
| 对书的姿势 | 评估者 | 找武器的人 |
| 核心问题 | 这本书在遮蔽什么 | 这本书里哪把武器我现在能用 |
| 输出物 | 关于书的洞察 | 关于自己问题的解法 |
| 批判是否触发 | 是，且是核心动作 | 否，只检验适用边界 |
| 参考人物 | 王沪宁 | 沈老师、毛主席 |

**切换信号**：这本书主要在传递可操作的方法和流程，而不是在建构一套"应该怎么想"的话语体系——用这个引擎。

---

## 零步：入场条件（不填不开书）

这是整个引擎最重要的一步。没有流血的具体问题，不开工具书。

```
【我现在的具体问题】：[一句话，具体到能感受到痛的程度。不是"我想学X领域"，是"我现在有X问题，它在Y场景里卡住了我，我需要一个工具来解决它"]

【为什么认为这本书有我需要的东西】：[一句话。不是"这本书评价很好"，是"这本书的第X章/核心工具，我认为和我的问题有直接关系，因为……"]

【我给自己的时间限制】：[几天内必须能提取到可以测试的东西，否则这本书不适合现在读]
```

**入场条件的作用**：它给了你一个过滤器。带着这个问题进书，批判模式没有空间启动，因为你的注意力被"这跟我的问题有没有关系"占满了。找武器的人不会在兵器库里发表武器设计的哲学批判。

---

## 一、快速定位（翻目录，不超过10分钟）

不从第一页开始读。先扫全貌，找出和你的问题最相关的章节。

```
【和我的问题直接相关的章节】：[列出，最多3章，不相关的跳过]

【核心工具/框架是什么】：[这本书的核心方法论是什么，一句话。不是主题，是方法]

【读完顺序】：[按相关性排，不按书的顺序]
```

**原则**：一本200页的工具书，和你的具体问题直接相关的通常不超过40页。找到那40页，其余的暂时不存在。

---

## 二、逐章提取（每章完成一次）

每章读完，强制填这个格式。填不进去，说明这章没读进去，或者这章和你的问题无关，跳过。

```
【这章的核心工具是什么】：[一句话，是一个可以操作的方法，不是一个观点或洞察]

【这个工具解决什么具体问题】：[工具的适用场景，越具体越好]

【这个工具失效的条件】：[在什么情况下这个工具不管用。这是边界检验，不是批判]

【和我的问题的关系】：
  直接命中 / 间接相关 / 无关（无关的直接丢掉）

【如果直接命中，我会怎么用】：[具体到下一个可执行的动作，有时间节点]
```

**关键**：批判不在这里。"这个工具的话语装置是X"不是这个格式需要的输出。如果批判冲动来了，记在旁边，读完再说。读书过程中批判是干扰，读完之后评估是正当的。

---

## 三、武器测试计划（读完一本书后填一次）

读完相关章节，填这个格式。这是让知识真正落地的关键步骤。

```
【我要测试的工具】：[从逐章提取里选出最直接命中的1-2个]

【测试场景】：[具体到哪个会议、哪个项目、哪个对话里测试]

【测试时间】：[具体日期，不是"尽快"，是下周三]

【成功的样子】：[可观察的、具体的结果。不是"感觉更好了"，是"X问题用这个工具处理之后，Y结果出现了"]

【失败的样子】：[工具失效的信号是什么，这样才能知道边界在哪里]
```

**原则**：没有测试计划的工具书学习，等于没学。工具只有在具体场景里被测试过，才真正进入你的认知库。测试失败不是浪费，是发现边界——这本身就是价值。

---

## 四、测试后复盘（测试完之后填）

```
【工具有效吗】：有效 / 部分有效 / 失效

【有效的条件】：[什么条件下这个工具表现好]

【失效的条件】：[什么条件下这个工具不管用]

【这个工具和我已有的哪个工具有关系】：[接入已有认知库。孤岛工具 = 没学会]

【我会在什么场景下再次使用它】：[有了真实经验之后，修正使用场景]
```

**这一步是沈老师路数的核心**：实例裁判循环。他不信任没有经过真实场景验证的认知。测试过的工具，边界是清晰的，可以放心用；没测试过的工具，是理论，不是武器。

---

## 五、两引擎的协作（有时候一本书需要两个引擎）

有些书既是工具书又是话语体系书，比如德鲁克——它有真实有效的工具，同时有隐性的控制功能。

这类书的读法：**先用工具书引擎提取，再用批判引擎检验**。顺序很重要，不能反过来。

先批判再提取的问题：批判完之后，你对这本书的怀疑已经建立，提取动力会下降，很多真正有用的工具会在怀疑的阴影里被忽略。

先提取再批判：你已经拿到了有用的武器，武器是你的了。然后你去看这把武器服务于什么利益结构，在什么边界外失效——这让你用武器更清醒，不让怀疑剥夺武器。

```
协作顺序：
① 判断这本书的类型（工具为主 / 话语体系为主 / 两者混合）
② 工具为主 → 直接用工具书引擎
③ 话语体系为主 → 直接用批判引擎
④ 两者混合 → 先工具引擎提取，完成测试计划后，再用批判引擎看话语功能
```

---

## 六、入场前的书型判断（30秒完成）

翻序言和目录，问三个问题：

**问题一**：这本书主要在告诉我"应该怎么想"，还是"怎么做某件具体的事"？
- 怎么想 → 批判引擎
- 怎么做 → 工具书引擎

**问题二**：这本书的结论依赖一个被隐藏的价值前提吗？
- 依赖隐藏前提 → 批判引擎
- 前提透明，只是方法论 → 工具书引擎

**问题三**：读完这本书，我期待产出的是洞察（关于这本书或这个领域的新理解），还是武器（可以直接部署到我当前问题里的工具）？
- 洞察 → 批判引擎
- 武器 → 工具书引擎

三个问题答完，引擎选择通常是清楚的。如果还不清楚，默认先用工具书引擎，批判留到读完之后。

## 附：Solomon进入工具书时的常见失格信号

- 翻开书第一反应是找可疑的句子（批判引擎在工具书场景里误触发）
- 读完一章，能说出"这章的核心观点"但说不出"这章的核心工具"（还在话语分析模式）
- 读完整本书，没有一个具体的测试计划（提取完成了但没有落地）
- 对书里某个工具说"这个有点道理"然后继续读（没有提取，只有评价）
- 读完之后的笔记都是关于这本书的，没有关于自己问题的（洞察没有转化为武器）
