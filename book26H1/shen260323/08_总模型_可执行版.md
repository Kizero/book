# 总模型：《卓有成效的管理者》可执行版

## 一、全书数据流图（10行以内）

```
输入：知识工作者 + 四个结构性约束（常数）
                    ↓
时间 ──整合为整块──> 有效工作的前提
                    ↓
贡献方向 ──确定接口──> 时间用在正确方向
                    ↓
长处配置 ──比较优势──> 相同时间，最大输出
                    ↓
要事删除 ──显式GC───> 资源集中于少数高价值项
                    ↓
决策质量 ──状态机───> 假说→检验→推行→反馈
                    ↓
输出：管理者.有效性 = true（可持续，不是偶发）
```

五步是依赖链，不是并列菜单。没有整块时间，决策质量无从谈起。

## 二、触发条件矩阵

| IF（触发） | THEN（行动） | 所属维度 |
|-----------|------------|---------|
| 感到忙碌但成果模糊 | 问：这件事完成后，外部系统会有什么不同？说不出来 → 停下来重新定义工作 | 有效性 |
| 每天深度工作 < 90分钟 | 找浪费来源（会议/信息/重复/配置），针对来源修复，不是靠意志力 | 时间 |
| 任何工作开始前 | 问：完成后外部世界有什么不同？（定义接口先于实现） | 贡献 |
| 做了与宣称标准相悖的决定 | 意识到：你刚刚更新了实际标准，不是说的那个 | 贡献→标准 |
| 要给人分配工作 | 问：谁在这里的delta最大？（比较优势，不是"谁不忙" | 长处 |
| 要设计一个职位 | 先写工作要求（接口），再找人（实现），不能反过来 | 长处 |
| 工作清单超过5项 | 放弃审计：如果今天不存在这项，我会重新启动吗？NO → free() | 要事 |
| 课程只增不减多年了 | 零基础重设计：先定义学生需要能做什么，再逐项justify现有内容 | 要事→GC |
| 遇到反复出现的问题 | 这是经常性问题，对着结构动作（建立通则），不是对着信号动作 | 决策要素1 |
| 只听到赞成声音 | 指定adversarial testing角色，要求书面反驳 | 决策 |
| 准备行动 | 先问：什么都不做的代价可以接受吗？可以 → 不做 | 决策 |
| 做出了决策 | 必须能回答：谁执行？他有能力资源吗？不能回答 → 决策未完成 | 推行 |

## 三、这个框架在哪里对我不适用

**高频、低成本的教学决策**：

德鲁克的框架适合"低频、高成本、不可逆"的决策。但我每天面对的大量教学决策是高频低成本的：这道例题用A还是B？现在要不要暂停问一下学生理解了没？课堂上临时出现的问题，现在回答还是留到课后？

把德鲁克的五要素状态机搬到这些决策上，成本远高于收益。这里需要更轻量的启发式规则，不是决策框架。

**这个框架的正确使用边界**：
- 低频（每学期或更少频次）
- 高成本（课程设计、教学方式重大调整、学生评价体系改变）
- 后果不可逆（发出去的评分、公开的教学承诺）

## 四、对我作为CS教育者的一个行为检验

**每周问自己的一个问题**：

> 这周我反复回答了哪类学生问题？这类问题反复出现，说明它是经常性问题。我对着信号动作了几次（个别回答）？我对着结构动作了吗（改进课程讲授方式）？

这一个问题覆盖了德鲁克最核心的洞察：经常性问题需要结构性解法，而不是每次重新处理的临时答案。

## 五、框架的CS同构总结

| 德鲁克概念 | CS对应 |
|-----------|--------|
| 有效性（Effectiveness） | Fitness for purpose（规格说明本身是对的） |
| 效率（Efficiency） | Correctness（符合规格说明） |
| 整块时间 | Batch processing（高setup cost任务用批处理） |
| 时间碎片的代价 | Cache miss cost（context switch代价） |
| 贡献导向 | Interface-first design（先定义接口再实现） |
| 比较优势 | Assignment problem的最优解（最大化总产出） |
| 放弃昨天 | Explicit garbage collection（显式释放死引用） |
| 要事专注 | 单线程执行（消除thread safety问题） |
| 经常性问题→建立通则 | O(n) → O(1)：构建lookup table |
| 从意见开始 | TDD：先写测试，再写实现 |
| 反面意见机制 | Adversarial testing / red teaming |
| 五要素决策 | 状态机（State machine） |
| 职位设计 | 依赖倒置原则（DIP） |

## 六、一个核心警告

忙碌是有效性最大的幻觉，对教师尤其如此。

批改了30份作业，备了3小时的课，回答了20个学生问题——这些都是efficiency指标。有效性问的是：学生的能力有什么变化？他们毕业后能做什么？如果这些指标回答不了那两个问题，那我只是在高效地做错的事。
