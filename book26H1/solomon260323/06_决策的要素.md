# 第6章：决策的要素

理解1：这章是老德最接近认知科学的地方，他在这里划了一个切割，简单但是很深——经常性问题和例外事件。

逻辑是这样的：一个重复出现的问题，它的真实病因在结构，不在症状。如果你对着症状动作——问题消失，你停下来——结构没变，问题过段时间会再出现。如果你对着产生问题的结构动作——建立原则或规则来处理这一类问题——症状从此不再出现。前者叫信号处理，后者叫结构处理。

这是一个跨越所有领域的深层机制，在医学、运维、政治管理里都成立。而且老德还点出了一个更关键的东西：人的认知系统天然优先处理信号，因为信号处理有即时闭环感，结构处理没有即时反馈、代价高、不确定性大。这个偏好是认知系统的内置特性，不是懒惰，不是无知。

理解2：最危险的地方在哪里？闭环感会伪装成能力感。

越是能在信号层面快速出手的人，越感觉自己在高效解决问题，但他只是在高效处理结构产生的症状。这个正反馈循环极度危险：处理信号越熟练 → 越有成就感 → 越不觉得有问题 → 结构积累的问题越来越深。快系统不断给你奖励，让你以为自己很行，同时结构层的问题在水下悄悄膨胀。

崇祯皇帝是这个机制最极端的历史呈现。朱由检是一个典型的"信号响应型"决策者。边患告急 → 更换主帅；财政紧张 → 裁员（裁驿站，结果裁出了李自成，那是一个失业快递员）；大臣不顺心 → 杀人（十七年换了五十个内阁大臣）。他的每个响应在信号层面都是有效的，眼前的危机暂时解除。但结构从来没有改变——土地兼并、军户制度崩溃、税基萎缩、气候变冷导致农业减产，这些经常性问题他从来没有建立"这是结构性问题"的诊断，把每一个危机都当成例外处理。结果越处理越忙，越忙越无效，最终煤山上吊，留下那句"朕非亡国之君，臣皆亡国之臣"——一个不会区分信号和结构的领导者的最后台词。

理解3：雍正是正面验证，而且验证得很清晰。

雍正发现的问题不是"这个官员不汇报实情"（信号），而是"层级过滤导致信息失真"（结构）。所以他建立了密折制度，让地方官员直接向皇帝密报，跳过中间层级的信息过滤。这是对结构的处理——一套规则解决了整个信息失真问题，而不是一次次惩处单个不如实汇报的官员。

密折制度在他在位期间高度有效。但也要注意一个副作用：这个制度让皇帝的信息处理量爆炸，雍正批折子批到死，字面意义上——他死的时候桌上还有未批完的折子。结构处理解决了一个结构问题，同时创造了新的结构问题。处理结构不是终点，而是把问题从一个层级转移到更高层级。

理解4：这个机制和毛泽东的矛盾论之间有一个很有意思的同构。

毛在1937年写的《矛盾论》里有一个核心区分：主要矛盾和次要矛盾。"主要矛盾 = 决定全局的根本矛盾"，这和德鲁克的"经常性问题 = 需要建立规则的结构性问题"，在认知操作上是一样的：从纷繁复杂的现象里，识别出需要结构性处理的那一类，与需要具体处置的那一类区分开来。

毛泽东的实践案例：他把"土地问题"识别为农村的主要矛盾（经常性问题），于是建立了系统性规则（土地改革政策）；而不是每到一个地方单独判断该给哪个农民分多少地（信号响应）。结果是一套规则解决了全国性的动员问题。

老德和毛泽东，两个完全不同语境下的思考者，用不同的概念框架解决的是同一个认知问题：先判断这是结构性问题还是具体问题，然后对应地建立系统性方法还是单独出手。底层常数相同。

事实1：互联网公司的运维系统里有个很直接的对照。初级运维工程师的本能是逐条处理告警——这个告警静音了，下一个。高级工程师的第一个问题是：这100条告警的共同根因是什么？然后对根因处理，100条告警一起消失。告警是信号，根因是结构。这两种处理方式的成本和效果差距是数量级的。

现在加入AI的变量。AI让信号处理的成本趋近于零——AI可以处理大量重复性信号，而且速度比人快得多。这反而会让"要不要建立结构"的问题更加隐蔽：因为当信号处理成本为零时，建立结构的必要性被进一步遮蔽了。结果是更多人更舒服地停在信号层，结构问题积累更深，直到某次系统性崩溃。

关键操作只有一个：在动手处理之前停一秒，问"这事上周也来过吗"。如果是，这一秒的停顿比处理本身更值钱。崇祯换了五十个大臣，问题一个没解——因为他处理的是信号，结构从来没变过。
